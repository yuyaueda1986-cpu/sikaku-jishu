# JSONファイル定義マニュアル

クイズデータを追加・編集する際の手順と形式を説明する。

---

## ファイル構成

```
data/
├── index.json       ← 試験一覧（必須、1ファイルのみ）
├── 2025-1.json      ← 2025年度第1回の問題データ
├── 2025-2.json      ← 2025年度第2回の問題データ
└── 2024-1.json      ← 2024年度第1回の問題データ（以下同様）
```

---

## 1. index.json（試験一覧インデックス）

アプリ起動時に最初に読み込まれるファイル。ホーム画面の試験一覧はこのファイルから生成される。

### 形式

```json
{
  "exams": [
    {
      "year": 2025,
      "round": 1,
      "kind": "電気通信主任技術者.法規",
      "label": "2025年度第1回 電気通信主任技術者法規",
      "file": "2025-1.json"
    },
    {
      "year": 2025,
      "round": 2,
      "kind": "電気通信主任技術者.電気通信システム",
      "label": "2025年度第2回 電気通信主任技術者電気通信システム",
      "file": "2025-2.json"
    },
    {
      "year": 2024,
      "round": 1,
      "kind": "電気通信主任技術者.法規",
      "label": "2024年度第1回 電気通信主任技術者法規",
      "file": "2024-1.json"
    }
  ]
}
```

### フィールド説明

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `year` | 整数 | ✓ | 試験の実施年度 |
| `round` | 整数 | ✓ | 試験の回次（1 or 2） |
| `kind` | 文字列 | ✓ | 試験種別（ドット区切り。例: `電気通信主任技術者.法規`） |
| `label` | 文字列 | ✓ | ホーム画面に表示されるラベル（`kind`を反映した表記） |
| `file` | 文字列 | ✓ | `data/` フォルダ内のファイル名 |

### 注意事項

- `file` に記載したファイルが `data/` フォルダに存在しない場合、選択時にエラーが表示される
- 一覧の表示順は `exams` 配列の順序に従う（新しい年度を先頭に置くことを推奨）
- ファイルを追加しただけでは表示されない。**必ず `index.json` への追記も行うこと**

## 2. YYYY-N.json（試験ごとの問題データ）

ファイル名は `index.json` の `file` フィールドと一致させること。

### 形式

```json
{
  "year": 2025,
  "round": 1,
  "kind": "電気通信主任技術者.法規",
  "label": "2025年度第1回 電気通信主任技術者法規",
  "questions": [
    {
      "id": 1,
      "text": "電気通信事業法において、電気通信事業者が提供する電気通信役務の料金その他の提供条件について、利用者に対して説明しなければならないとされているのは、次のうちどれか。",
      "choices": [
        "契約締結前の説明義務",
        "契約締結後の報告義務",
        "年次の利用状況報告義務",
        "四半期ごとの料金通知義務"
      ],
      "correctIndex": 0,
      "explanation": "電気通信事業法第26条では、電気通信事業者は契約の締結前に、料金その他の提供条件について利用者に説明しなければならないと規定されています。",
      "aiPromptTemplate": "電気通信事業法における「契約締結前の説明義務」について、第26条の具体的な内容と、事業者が説明すべき事項を詳しく解説してください。"
    }
  ]
}
```

### ルートフィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `year` | 整数 | ✓ | 試験の実施年度 |
| `round` | 整数 | ✓ | 試験の回次 |
| `kind` | 文字列 | ✓ | 試験種別（ドット区切り。例: `電気通信主任技術者.法規`） |
| `label` | 文字列 | ✓ | 試験の表示ラベル（`kind`を反映した表記） |
| `questions` | 配列 | ✓ | 問題の配列 |

### questions 各要素のフィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | 整数 | ✓ | 問題ID（1始まりの連番を推奨） |
| `text` | 文字列 | ✓ | 問題文 |
| `choices` | 文字列配列 | ✓ | 選択肢（1〜16個） |
| `correctIndex` | 整数 | ✓ | 正解の選択肢インデックス（0始まり） |
| `explanation` | 文字列 | ✓ | 解説テキスト |
| `aiPromptTemplate` | 文字列 | ✓ | AI向けプロンプトテキスト |
| `figure` | オブジェクト | | 図表情報（省略可） |

### figure オブジェクトのフィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | 文字列 | ✓ | 図の種類（`"mermaid"`, `"svg"`, `"png"` のいずれか） |
| `content` | 文字列 | ※ | Mermaid記法テキスト（`type` が `"mermaid"` の場合に必須） |
| `src` | 文字列 | ※ | 画像ファイル名（`type` が `"svg"` または `"png"` の場合に必須。`data/images/` 配下） |
| `alt` | 文字列 | | アクセシビリティ用代替テキスト（省略時は空文字） |

### figure の使用例

**Mermaid図表:**

```json
{
  "id": 1,
  "text": "次のネットワーク構成図について正しいものを選べ。",
  "choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  "correctIndex": 0,
  "explanation": "解説テキスト",
  "aiPromptTemplate": "AIプロンプト",
  "figure": {
    "type": "mermaid",
    "content": "graph TD;\n  A[ルータ] --> B[スイッチ];\n  B --> C[端末1];\n  B --> D[端末2];",
    "alt": "ネットワーク構成図"
  }
}
```

**SVG画像:**

```json
{
  "figure": {
    "type": "svg",
    "src": "network-diagram.svg",
    "alt": "ネットワーク構成図"
  }
}
```

**PNG画像:**

```json
{
  "figure": {
    "type": "png",
    "src": "circuit-diagram.png",
    "alt": "回路図"
  }
}
```

## 3. correctIndex の指定方法

`correctIndex` は `choices` 配列の **0始まりのインデックス**。

```json
"choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
"correctIndex": 0   → 正解は「選択肢A」
"correctIndex": 2   → 正解は「選択肢C」
```

選択肢ランダム化が有効な場合、QuizEngine が `correctIndex` を自動的に追従するため、JSONは常に **元の配列順での正解インデックス** を記載する。

---

## 4. aiPromptTemplate の書き方

AI解説誘導テキストはユーザーがコピーして ChatGPT 等に貼り付けるプロンプト。以下の点を意識して書く。

- **具体的なキーワードを含める**（条文番号、専門用語など）
- **質問形式**にする（「〜を解説してください」「〜の違いを教えてください」）
- 短くまとめる（100文字程度が目安）

**例（良い例）:**

```
電気通信事業法第45条に基づく「工事、維持及び運用の監督」の具体的な内容と責任範囲を詳しく解説してください。
```

**例（悪い例）:**

```
この問題について教えてください。  ← 情報が少なすぎる
```

## 5. 新しい試験を追加する手順

1. `data/YYYY-N.json` を作成し、問題データを記入する
2. `data/index.json` の `exams` 配列に以下を追記する

```json
{
  "year": 2026,
  "round": 1,
  "kind": "電気通信主任技術者.法規",
  "label": "2026年度第1回 電気通信主任技術者法規",
  "file": "2026-1.json"
}
```

3. ブラウザでリロードし、ホーム画面に新しい試験が表示されることを確認する

## 6. バリデーションチェックリスト

問題データを作成・編集した後、以下を確認する。

- [ ] `correctIndex` が `choices` の範囲内か（0 〜 `choices.length - 1`）
- [ ] `choices` が1〜16個か
- [ ] `id` が重複していないか
- [ ] `index.json` の `file` と実際のファイル名が一致しているか
- [ ] JSONとして構文エラーがないか（ブラウザの開発者ツールで確認）
- [ ] `aiPromptTemplate` が空でないか

---

## 7. よくあるミスと対処

| ミス | 症状 | 対処 |
|---|---|---|
| `index.json` への追記忘れ | ホームに試験が表示されない | `index.json` に `exams` エントリを追加 |
| `correctIndex` の範囲外 | ランダム化時に正解がずれる | `choices` の配列長を確認し修正 |
| JSONの末尾カンマ | データ読み込みエラー | 最後の要素の後ろにカンマをつけない |
| ファイル名のtypo | データ読み込みエラー | `index.json` の `file` 名とファイル名を照合 |
